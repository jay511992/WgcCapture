# GraphicsCapture 易语言声明表（32 位 / x86）

本文档供 **易语言 5.x / 6.x（32 位）** 调用 `GraphicsCapture.dll` 使用。

- 调用约定：**stdcall**
- 必须使用 **`out/x86/GraphicsCapture.dll`**（32 位版本）
- 易语言无 x64 时，**不要**使用 `out/x64` 下的 DLL

> 说明：易语言程序为 32 位，仍可传入 **64 位进程窗口** 的 `HWND` 进行截图（如 `Gw2-64.exe`），只要该窗口句柄有效且可见即可。

## 六、多线程截图同一窗口

**规则（必看）：**

- 多个线程截**同一个 HWND** 时，**不能**每个线程各 `Open` 一个 Session，也**不能**多线程同时 `GC_CaptureSessionFrame(同一个会话, ...)`。
- 推荐两种写法：
  - **方案 A**：全局许可证 + `GC_CaptureWindow`（实现简单，适合偶尔截图）
  - **方案 B**：一个专用截图线程持有一个 Session，其他线程只读共享的最新帧（适合高频、多线程分析）
  - **方案 C（10+ 线程）**：在方案 B 基础上加**帧版本号**，避免每个分析线程重复拷贝同一帧（见 6.3）
  - **方案 D（每线程不同识别区域）**：在方案 C 基础上，**只截一张全图**，各线程按自己的 `GC_Rect` 在内存里裁剪（见 6.4，**最推荐**）

以下示例假设目标窗口句柄保存在 **`目标窗口`**（整数型程序集变量）中。

---

### 6.3 方案 C：10+ 分析线程（版本号 + 批量启动）

**架构：**

```text
                    ┌─ 分析线程1 ─┐
                    ├─ 分析线程2 ─┤
[专用截图线程] ──→ 共享帧缓存 ──┼─ 分析线程3 ─┤  （10+ 个，只读，不调 DLL）
  （唯一调 DLL）    + 帧版本号    ├─   ...     ─┤
                    └─ 分析线程N ─┘
```

**原则：**

- 仍然只有 **1 个截图线程** 调用 `GC_OpenCaptureSession / Frame / Close`。
- 10+ 个分析线程**绝不**调用任何 `GC_Capture*` / `GC_Open*` / `GC_Close*`。

| 建议 | 说明 |
|------|------|
| 只用方案 C，不要用方案 A | 10+ 线程排队调 `GC_CaptureWindow` 会非常慢 |
| 截图线程 1 个即可 | 30fps 通常够用；再高注意 CPU/GPU |

### 6.4 方案 D：每线程识别区域不同（推荐）

**思路：** 截图线程只截 **一张全图（客户区）** 放入共享缓存；每个分析线程自带一个 `GC_Rect`（相对**客户区左上角**），只拷贝/识别自己的区域。

**不要**让每个线程各自调用 `GC_CaptureWindowEx(区域...)`——DLL 内部仍会先截整窗，反而更慢。

```text
[截图线程] ──→ 全图缓存 + 版本号（仅截图线程持有完整大图）
                  ├─ 线程1：复制最新区域Ex → 小图 220×40  → 识别
                  ├─ 线程2：复制最新区域Ex → 小图 200×200 → 识别
                  └─ 线程N：只拷自己的区域，不拷全图
```

**何时用多个截图 Session？（少数情况）**

仅当同时满足以下条件才考虑 2 个截图线程 + 2 个区域 Session：

- 两块区域都很大（例如各 800×600）
- 需要的刷新率不同（A 要 60fps，B 要 5fps）
- 区域在屏幕上相距很远

否则一律 **6.4 单 Session 全图 + 本地裁剪**。

| 写法 | 是否推荐 |
|------|----------|
| 1 截图线程全图 + `复制最新区域Ex` | ✅ **最推荐**（省内存） |
| 1 截图线程全图 + 拷全图再 `裁剪BGRA区域` | ⚠️ 线程少时可用 |
| 每线程各调 `GC_CaptureWindowEx(不同区域)` | ❌ 慢 |
| 每线程各 `OpenCaptureSession` 同一 HWND | ❌ 易出错 |

## 十、注意事项

1. **DLL 路径**：只用 `out\x86\GraphicsCapture.dll`，与易语言 EXE 放同一目录。  
2. **必须释放**：成功后调用 `GC_FreeImage` / `GC_FreeImageBgr24` / `清空GC图像`。  
3. **BGR24 推荐**：内存约为 BGRA 的 75%；像素顺序 B,G,R。  
4. **BGRA 字节序**：蓝绿红 Alpha，显示前可能需要交换 R/B。  
5. **64 位目标窗口**：易语言虽为 32 位，仍可截 64 位程序的窗口，需自行枚举得到 HWND。  
6. **GW2**：建议 `GC_CaptureWindowExBgr24` + `GC_FLAG_RENDER_FULL_CONTENT`（2），默认无标题栏。  
7. **花屏**：`GC_Image` / `GC_ImageBgr24` 均为 4 个整数型成员、共 16 字节。  
8. **多线程同一窗口**：见 [第六节](#六多线程截图同一窗口)；Session 不可跨线程共享。
